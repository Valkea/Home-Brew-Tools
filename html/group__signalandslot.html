Detailed Description

This group aggregates all classes related to the Signal & Slot library.

The signals and slots are systems made to dispatch on demand informations to registered callbacks.

Define Documentation

#define ENABLE_DELAYED_SIGNALS   false

This enables the delayed events.

If this is enabled, the Signal will accept delayed events and Signal will have the "operator( delay, args... )" enabled.
Defined in Signal.h

Warning:
The cost of this, is that Signal is 20 bytes with delayed events, and 8 bytes without them.
#define ENABLE_FORCE_SYNC   true

This forces delayed events to synchronize on the expected time instead of real time.

This way, if for any reason there is a big "lag" between two SignalManager updates, it will try to register and fire already passed delayed events until the current time is reached. 

Example: Suppose that a function initializes a delayed event to call itself 1 sec later. If there is a "big" lag of say 4.5 sec, the function will fire 4 times before the next update.

If this side effect is not required, ENABLE_FORCE_SYNC_NO_REWIND should be used.

Defined in Signal.h

Warning:
This define is useful ONLY when a delayed event calls a function that initializes (a) delayed event(s).
This require ENABLE_DELAYED_SIGNALS to be enabled in Signal.h
#define ENABLE_FORCE_SYNC_NO_REWIND   true

This disables the synchronized delayed events ability to register in past time.

This way, as ENABLE_FORCE_SYNC is enabled, it will try to lower at maximum the lost time, but will never register lower than present time. 

Example: Suppose that a function initializes a delayed event to call itself 1 sec later. If there is a "big" lag of say 4.5 sec, the function will fire 1 time before the next update and will initialize the next one as soon as possible (present time + 1 ms)

Defined in Signal.h

Warning:
This define is useful ONLY when a delayed event calls a function that initializes (a) delayed event(s).
This require ENABLE_DELAYED_SIGNALS and ENABLE_FORCE_SYNC to be enabled in Signal.h
#define ENABLE_DELEGATE_LIFECYCLES   true

This enables the life cycles on the Delegates.

If this is enabled, the "lifeCycles" parameter on the "connect" methods is taken into account, thus enabling the connected functor, function or method to be automatically disconnected from the Signal when the defined cycle number is reached.

Defined in Signal.h

Warning:
The current maximum value for lifeCycles of a delegate is 65535 (U16), if you need more than that, you need to change the "lifeCycles" variables in the Signal.h file from U16 to U32 (or U64), and to do the same with the 3rd value of the "T_EventHandler" typedef.
