Detailed Description

This group aggregates all classes related to the pout library.

Print-Streams are kinds of std::ostream (as cout and cerr) with verbosity and channels.

Define Documentation

#define ENABLE_PRINTMGR_LOG   true

This enables the PrintManager log messages to be compiled.

This option offers the ability to easily disable the PrintManager internal log system at release. By disabling it, the LOG_TO_FILE calls will be removed at compile time, thus avoiding useless function calls at runtime without having to remove by hands the LogToFile() calls everywhere. 

This SHOULD be disabled at release
Warning:
LOG_TO_FILE() can't be used outside of PrintManager itself as it refers to the private LogToFile() method.
See also:
PrintManger.setEnableLog to enable/disable the PrintManager log at runtime.
#define AUTO_INIT_PRINTMGR   true

This enables hbt::printManager to auto-initialize as a reference to the PrintManager singleton.

Thus a global alias of the PrintManager singleton exists for convenience and for optimization*, but if a manual initialization is required/preferred, this define should be turned to false, and the PrintManager singleton manually initialized.
Defined in PrintManager.h 

*It calls the static method once and redirects all incoming hbt::printManager calls directly to the PrintManager singleton instead of asking for the singleton reference each time PrintManager::getInstance() is called.

#define POUT_TO_DEBUG   true

This enables the POUT_DEBUG() calls to use some hbt::pout calls in debug mode only.

This option offers the ability to add an hbt::pout witch can be easily removed at release. 

 POUT_DEBUG( verbo, channel, msg );

can be used instead of 

 hbt::pout( verbo, channel ) << msg;

By disabling this option, the POUT_DEBUG calls will be removed at compile time, thus avoiding useless function calls at runtime without having to remove by hands the hbt::pout calls everywhere.

#define ENABLE_AUTO_DISCONNECT_PM   true

This enables the PrintToFiles to unregister themselves from the PrintManager.

If this is enabled, the PrintToFile will unregister from the PrintManager when the destructor is called. This way, it's not needed anymore to inform the connected PrintManager about the PrintToFile destruction.
